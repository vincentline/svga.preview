## 1. 核心原则

**执行规则**：任务描述不清、不可行或存在风险 → 立即停止执行 → 要求用户修改

**简化原则**：减少填写负担，只要求必要信息，不做过度规范要求

## 2. 不合规任务类型

| 类型 | 具体情况 | 处理方式 |
|------|---------|----------|
| **描述不清** | 表达模糊、前后矛盾、现状描述与实际不符 | 停止执行，要求澄清 |
| **不可行** | 技术无法实现、超出能力范围 | 停止执行，说明原因 |
| **风险问题** | 违反法律法规、存在安全风险 | 停止执行，拒绝处理 |
| **工作量过大** | 任务范围过大、工作量巨大 | 停止执行，要求拆分 |

## 3. 任务提交基本要求

### 3.1 必要信息

- **任务标题**：简洁明了，能反映任务内容
- **任务类型**：Bug修复、新功能、优化、文档更新等
- **问题描述**：说明需要解决的问题或改进的地方
- **预期结果**：任务完成后希望达到的效果

### 3.2 可选信息

- **关联文件**：涉及的文件路径（如有）
- **测试方法**：验证任务完成的方法（如有）
- **技术要求**：特定的技术实现要求（如有）

## 4. 不合规任务处理流程

1. **快速检查**：收到任务后，快速检查是否存在不合规情况

2. **立即停止**：发现任何不合规情况，立即停止任务执行

3. **明确反馈**：向用户说明具体的不合规原因
   - 例如："任务描述前后矛盾，无法确定具体需求，请澄清"
   - 例如："该需求存在安全风险，无法执行"

4. **要求修改**：明确告知用户需要如何修改才能符合要求

5. **重新评估**：用户修改后，重新检查是否符合要求

6. **确认执行**：完全符合要求后，才开始正式执行任务

## 5. 示例任务格式

### 5.1 正确示例

```
# 任务标题：修复SVGA播放器加载失败问题

## 任务信息
- 任务类型：Bug修复
- 关联文件：docs/assets/js/core/app.js

## 任务描述
- 问题：大型SVGA文件（超过10MB）加载时会失败
- 预期结果：能够正常加载和播放大型SVGA文件
```

### 5.2 不合规示例及原因

#### 示例1：描述不清
```
# 任务标题：修复bug

## 任务信息
- 任务类型：Bug修复

## 任务描述
- 问题：有个bug需要修复
- 预期结果：修复好bug
```
**原因**：描述过于模糊，无法确定具体问题

#### 示例2：工作量过大
```
# 任务标题：重构整个项目

## 任务信息
- 任务类型：优化

## 任务描述
- 问题：项目代码质量差
- 预期结果：完全重构项目，提升代码质量
```
**原因**：任务范围过大，工作量巨大，需要拆分

#### 示例3：不可行
```
# 任务标题：实现AI自动生成SVGA

## 任务信息
- 任务类型：新功能

## 任务描述
- 问题：需要AI自动生成SVGA动画
- 预期结果：上传图片后自动生成完整的SVGA动画
```
**原因**：技术复杂度高，当前技术栈难以实现

## 6. 附则

- 本规则适用于所有提交的任务
- 任务处理人员有权根据本规则判断任务是否合规
- 本规则将根据实际执行情况持续优化
