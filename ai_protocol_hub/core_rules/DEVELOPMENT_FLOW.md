# * 收尾工作流

## 触发方式
当收到【收尾】指令时，执行以下收尾流程：

## 收尾流程

### 1. 代码精简与优化
- 删除冗余代码和重复逻辑
- 优化函数和类的结构，提高可读性
- 检查并修复代码中的性能问题

### 2. 删除调试代码
- 移除所有 `console.log`、`debugger` 等调试语句
- 删除用于测试的临时变量和函数
- 移除开发环境特有的代码

### 3. 代码规范检查
- 确保所有函数、类和模块都有完整的中文注释
- 检查并更新过时的注释
- 统一注释风格，符合项目规范
- 检查代码缩进和命名规范
- 确保所有文件都遵循项目的代码规范
- 运行代码检查工具，修复语法错误

### 4. 更新文档
- 确定当前时间：使用`ai_protocol_hub/scripts/GET_TIME.ps1`中的PowerShell命令获得的时间
- 通过GitHub查看最近的提交记录
  - 检查新更新记录中是否有未添加到索引的功能项，有的话更新 `INDEX.md` 索引文档，确保所有功能都已正确索引
  - 检查新更新记录中是否有功能新增、核心逻辑修改或技术栈变更未添加到 `README.md` 文档，有的话更新 `README.md` 文档
  - 将文档中的「最后更新时间」修改为上面确定的当前时间
- 检查并更新其他相关文档
