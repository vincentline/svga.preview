# 测试规则

## 1. 测试工具选择
- 优先使用IDE内置测试工具
- 无内置工具时，使用webapp-testing技能进行测试

## 2. 浏览器模式
- 禁止使用无头浏览器模式
- 使用browser-use时，必须添加 `--browser real` 参数

## 3. 测试准备
- 测试前查找项目内可用的测试模板、测试用例、测试流程
- 参考现有测试资源进行测试，避免重复工作

## 4. 测试后处理
- 若用户反馈问题已解决：
  1. 测试过程中添加的冗余调试代码，若与问题原因无关且未解决问题，测试后需删除或还原
  2. 删除冗余调试代码
  3. 在webapp-testing技能中新建或优化相关测试模板
  4. 执行【提交GitHub】操作
  5. 提醒用户整理经验文档

## 5. 执行流程
1. 检查项目是否有内置测试工具
2. 查找并参考现有测试资源
3. 选择合适的测试工具开始测试
4. 测试完成后进行相应的后处理操作
