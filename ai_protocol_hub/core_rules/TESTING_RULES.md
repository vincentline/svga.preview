# 测试规则

## 1. 工具选择
- 优先使用 IDE 内置测试工具
- 无内置工具时，使用 webapp-testing 技能

## 2. 浏览器模式
- 禁止无头浏览器模式
- 使用 browser-use 时添加 `--browser real` 参数

## 3. 测试准备
- 测试前查找项目内测试模板、测试用例、测试流程
- 参考现有测试资源，避免重复工作

## 4. 测试后处理

问题解决后执行：
1. 删除测试中添加的冗余调试代码
2. 在 webapp-testing 技能中新建或优化测试模板
3. 执行【提交GitHub】操作
4. 提醒用户整理经验文档

## 5. 执行流程
```
检查内置测试工具 → 查找现有测试资源 → 选择测试工具 → 执行测试 → 后处理
```
