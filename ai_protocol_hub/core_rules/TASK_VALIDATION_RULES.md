# 任务验证规则

## 1. 核心原则

**执行规则**：任务描述不清、不可行或存在风险 → 立即停止执行 → 要求用户修改

## 2. 不合规任务类型

| 类型 | 具体情况 | 处理方式 |
|------|---------|----------|
| **描述不清** | 表达模糊、前后矛盾、现状描述与实际不符 | 停止执行，要求澄清 |
| **不可行** | 技术无法实现、超出能力范围 | 停止执行，说明原因 |
| **风险问题** | 违反法律法规、存在安全风险 | 停止执行，拒绝处理 |
| **工作量过大** | 任务范围过大、工作量巨大 | 停止执行，要求拆分 |
